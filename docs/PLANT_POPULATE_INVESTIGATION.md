# Plant Populate Investigation

## Scope

This is a P2 populate-clutter line for `LargePlantContainerFactory` and
related plant container assets. It includes optional timing, targeted
benchmarks, and one default-off Wheat-only template reuse experiment behind:

```bash
INFINIGEN_REUSE_PLANT_TEMPLATE_GEOMETRY=1
```

The experiment does not simplify leaves or stems, does not reduce clutter,
does not change solver behavior, and does not add concurrency or C++. It does
change internal Wheat random consumption on cache hits because the original raw
leaf/stem geometry generation is skipped. The stable Isaac Sim static script
`scripts/run_isaac_static_optimized_10room.sh` is unchanged.

## Source Path

`PlantContainerFactory` and `LargePlantContainerFactory` are defined in:

```text
infinigen/assets/objects/tableware/plant_container.py
```

`LargePlantContainerFactory` forces:

```text
plant_factories = [MonocotFactory]
```

`MonocotFactory` is defined in:

```text
infinigen/assets/objects/monocot/generate.py
```

It wraps grass and non-grass monocot growth factories from:

```text
infinigen/assets/objects/monocot/
```

Important concrete monocot factories include:

```text
AgaveMonocotFactory
BananaMonocotFactory
TaroMonocotFactory
GrassesMonocotFactory
WheatMonocotFactory
MaizeMonocotFactory
TussockMonocotFactory
VeratrumMonocotFactory
```

Smaller plant containers can also wrap factories in:

```text
infinigen/assets/objects/small_plants/
```

## Call Chain

Final populate path:

```text
populate_state_placeholders(final=True)
  os.generator.spawn_asset(i=inst_seed, loc=placeholder.location, rot=...)
    AssetFactory.spawn_asset()
      LargePlantContainerFactory.create_placeholder()
      PlantContainerFactory.create_asset()
        PlantPotFactory.create_asset()
        read pot edge geometry and separate dirt surface
        PlantPotFactory.finalize_assets()
        fill/subsurf dirt mesh
        dirt_surface.apply(dirt_)
        apply_modifiers(dirt_)
        remove dirt vertices outside pot radius
        MonocotFactory.spawn_asset()
          concrete monocot factory create_asset()
          optional join_objects(monocots)
        origin2lowest(plant)
        MonocotFactory.finalize_assets()
        scale and place plant into pot
        join_objects([pot, plant, dirt])
```

`plant_spawn_duration` now corresponds to:

```text
self.plant_factory.spawn_asset(i=i, loc=(0, 0, 0), rot=(0, 0, 0))
origin2lowest(plant, approximate=True)
```

For `LargePlantContainerFactory`, that enters:

```text
MonocotFactory.spawn_asset()
  AssetFactory.spawn_asset()
    MonocotFactory.create_asset()
      concrete_monocot_factory.create_asset()
        create_raw()
          make_collection()
            build_instance()
              build_leaf()
          build_stem()
          surface.add_geomod(..., apply=True)
        optional branch / ear / husk factory create_asset()
        decorate_monocot()
```

## Likely Slow Points

- `plant_factory.spawn_asset()`: this is the largest measured substage in the
  first benchmark and includes concrete monocot geometry generation.
- Leaf / stem geometry generation in monocot factories, often using curves,
  geometry nodes, modifiers, and joins.
- Branch / ear / husk geometry for `WheatMonocotFactory`,
  `VeratrumMonocotFactory`, `TaroMonocotFactory`, `MaizeMonocotFactory`, and
  related factories.
- Dirt mesh editing, grid fill, subdivision, material assignment, and modifier
  application.
- Large numbers of temporary meshes / objects generated by plant leaves,
  branches, or clustered grass monocots.
- Geometry node groups with `singleton=False` in several plant implementations.
- Pot / dirt materials are visible but are not the dominant measured cost in
  the latest benchmark.

## Timing

Optional timing is enabled with:

```bash
INFINIGEN_PROFILE_PLANT_ASSETS=1
```

CSV path:

```text
<output_folder>/infinigen_plant_assets_timing.csv
```

Fallback path:

```text
/tmp/infinigen_plant_assets_timing.csv
```

Targeted runs can override the path with:

```bash
INFINIGEN_PLANT_ASSETS_TIMING_CSV=/path/to/infinigen_plant_assets_timing.csv
```

The timing records factory class, wrapped plant factory class, concrete
monocot factory class, pot factory class, total duration, container duration,
geometry duration, material duration, pot create / finalize duration, dirt
geometry / material duration, plant spawn / finalize / placement duration,
leaf / stem / branch generation duration when safely observable, leaf / stem /
branch call counts, leaf / stem / branch mesh churn, geometry template
candidate key, heuristic geometry reuse risk level, Plant template reuse
enabled/used flags, cache hit/miss counts, cache key, cache size, reuse scope,
fallback count, modifier apply duration, join duration, before/after datablock
counts, created datablock counts, material and node-group prefix summaries,
success, and error type.

Some low-level work cannot be safely separated yet. In particular,
`nodegroup_generation_duration` remains `0` in the current timing because
node-group creation is buried inside geometry-node helper calls; the CSV still
records created node-group counts and prefix summaries for attribution.

Analyze with:

```bash
python scripts/analyze_plant_assets_timing.py \
  outputs/bench_plant_assets/infinigen_plant_assets_timing.csv
```

## Targeted Microbenchmark

Added:

```text
scripts/bench_plant_assets_factory.py
```

Default:

```bash
python scripts/bench_plant_assets_factory.py \
  --samples 30 \
  --seed 0 \
  --output_folder outputs/bench_plant_assets
```

Filter to concrete monocot factories:

```bash
python scripts/bench_plant_assets_factory.py \
  --samples 50 \
  --seed 0 \
  --concrete-plant-filter WheatMonocotFactory,GrassesMonocotFactory \
  --output_folder outputs/bench_plant_assets_wheat_grasses
```

The benchmark instantiates `LargePlantContainerFactory(factory_seed)`, creates
a placeholder, calls `spawn_asset(inst_seed, placeholder=placeholder)`, records
timing, and deletes generated objects. It is not a full-scene walltime result.

The benchmark bounds its random `factory_seed` range by default to avoid an
existing nested `FixedSeed` overflow in some `Veratrum` branch-factory seeds.
This only affects the targeted benchmark sampling range and does not change
real scene generation.

## Reuse Suitability

Potential later opt-in candidates:

- Narrow material template reuse for dirt or simple plant shaders if random
  color and parameter variation can be preserved. The latest benchmark does
  not make this the first priority.
- Fixed helper node group reuse only after confirming the node group does not
  embed per-instance leaf/stem shape, material, tag, or object references.
- Concrete monocot subfactory profiling before any plant-wide reuse change.
- A narrow opt-in plant template geometry experiment for repeated leaf/stem
  helper geometry, but only after visual-risk review. The likely switch name
  for such an experiment is:

```bash
INFINIGEN_REUSE_PLANT_TEMPLATE_GEOMETRY=1
```

High-risk or unsuitable first changes:

- Reusing full plant meshes or plant templates, because that can visibly
  repeat leaf/stem silhouettes and reduce scene complexity.
- Simplifying leaves, branch counts, geometry-node output, or subdivision as a
  speed path without a quality gate.
- Reusing top-level geometry nodes that carry object-specific inputs.
- Any change that skips random sampling in the concrete plant factory.

## Next Opt-In Direction

The first speed lever now available for review is a very narrow,
default-off geometry/template experiment:

```bash
INFINIGEN_REUSE_PLANT_TEMPLATE_GEOMETRY=1
```

The implemented v1 starts with `WheatMonocotFactory` raw-mesh reuse only.
`GrassesMonocotFactory` remains the second candidate, but should wait until
Wheat passes visual review. Do not start with `VeratrumMonocotFactory` or
`AgaveMonocotFactory` because their branch systems and leaf deformation are
higher visual-risk sources of variation. It must not reduce plant count or
broad visual complexity. A lower-risk but likely smaller follow-up is:

```bash
INFINIGEN_REUSE_PLANT_NODEGROUPS=1
```

only for fixed helper node groups proven not to encode per-instance shape,
material, tag, or object references. `INFINIGEN_REUSE_PLANT_MATERIALS=1` is
not the first recommendation from the latest benchmark because material
generation was only about `0.5%` of measured time.

## Risk

Plants are visually complex and shape-driven. The main risks are reduced
leaf/stem variation, visibly repeated silhouettes, broken dirt/pot alignment,
bad material diversity, and accidental changes to random flow. Future plant
optimizations should stay opt-in and should start from concrete subfactory
evidence, not a broad wrapper-level shortcut.
